01 / ROOT

NO IdP. NO CLOUD. YOU RUN IT.

ROOT

vault42 is a self-hosted identity authority. A single Go binary issues every session JWT using RS256 under a strict algorithm whitelist, and publishes its public keys at a JWKS endpoint so any resource server verifies locally.

No external IdP. No cloud. You run it. Stateless access tokens, stateful refresh with replay-family revocation, PostgreSQL-backed with least-privilege roles. Embedded SPA or pure API.

  • One static binary — CGO_ENABLED=0, distroless, read-only fs.
  • RS256 / JWKS resource-server model — tokens verifiable by any service.
  • Full session lifecycle: register, login, refresh, logout, MFA.

  3. 02

    ISSUE RS256 JWTs

    Users register, log in, and refresh. vault42 mints short-lived access tokens signed RS256 under the active kid — stateless and tamper-evident.

  5. 03

04 / PROOF

SPECIFICS, NOT ADJECTIVES.

+
RS256 ONLY
Algorithm whitelist rejects none, HS256, ES*, and all others at parse time. Drops jku, jwk, x5u headers. 2048-bit RSA. 8 KB size cap.
+
JWKS ENDPOINT
GET /.well-known/jwks.json. Any resource server fetches public keys. Zero-downtime rotation. Retired keys kept for the validation window.
+
FIDO2 / WEBAUTHN
Passkey registration and authentication. Security keys + platform authenticators. Works alongside TOTP + backup codes.
+
APPEND-ONLY AUDIT
DB-level enforcement. PostgreSQL triggers block UPDATE/DELETE on audit_log. vault_app role restricted to INSERT + SELECT only.
+
REFRESH ROTATION
Single-use refresh tokens. Family tracking. Replay of a used token nukes the entire family. Device-fingerprint bound.
+
MINIMAL + HAND-ROLLED
Exactly three direct dependencies. JWT, TOTP, JWKS, fingerprint, constant-time, AES-GCM — stdlib or small internal packages.
+
RUNS ON A PI
Embedded profile: ~60–80 MB idle RAM, in-memory cache, 5 DB conns, auto-migrate. ARM64 build supported.
+
ARGON2ID + FINGERPRINT
46 MiB / 1 iter, NIST SP 800-63B. HIBP k-anonymity check. Every token carries a device-fingerprint hash.
